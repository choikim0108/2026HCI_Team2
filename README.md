# 2026 HCI Team 2

한림대학교 수강신청 UX 개선을 목표로 한 HCI 프로토타입 프로젝트입니다.
기존 수강신청 사이트의 사용성 문제를 분석하고, 탐색-비교-신청-확인 흐름을 더 직관적으로 재설계했습니다.

---

## 1. 프로젝트 개요

- 프로젝트명: Hallym Course Registration UX Prototype
- 저장소명: 2026HCI_Team2
- 성격: HCI 수업용 웹 프로토타입
- 개발 형태: 프론트엔드 단일 페이지 프로토타입
- 핵심 목표:
  - 수강신청 과정에서의 인지 부하 감소
  - 시간표 확인과 신청 의사결정의 즉시성 강화
  - 검색/필터/정보 구조를 개선해 탐색 효율 향상

---

## 2. 문제 정의 (기존 사이트 대비)

기존 수강신청 시스템을 사용하며 확인한 주요 한계는 다음과 같습니다.

1. 시간표와 실시간 연동 부재
	- 과목 선택 시 실제 시간표 배치와 충돌 여부를 즉시 파악하기 어려워, 신청 전후로 반복 확인이 필요했습니다.

2. 비실시간 검색 흐름
	- 검색어 입력 후 Enter 또는 버튼 입력이 필요해 조건을 바꿔 탐색할 때 반복 동작이 많고 흐름이 끊겼습니다.

3. 과도한 가로 정보 배치
	- 한 화면에 가로 방향 정보가 많이 펼쳐져 핵심 정보 비교가 어렵고 시선 이동 비용이 컸습니다.

4. 이수구분/구분 접근성 저하
	- 선택 항목 종류는 많지만 분류 방식이 직관적이지 않아 원하는 조건으로 빠르게 좁혀가기 어려웠습니다.

5. 상세 정보 확인 동선 비효율
	- 강의계획, 정원/대기, 영역별 이수 현황 확인이 분리되어 탐색-판단-신청의 연속성이 떨어졌습니다.

---

## 3. 개선 목표 및 설계 방향

1. 실시간 탐색
	- 입력 즉시 반영되는 검색으로 탐색 리듬을 유지합니다.

2. 신청과 시간표의 즉시 연결
	- 과목 신청/취소 상태가 시간표에 즉시 반영되도록 구성합니다.

3. 핵심 정보 우선 구조
	- 강좌 카드 중심으로 교수/학과/학점/시간/정원 정보를 빠르게 비교 가능하게 설계합니다.

4. 상태 가시성 강화
	- 정원 초과, 대기신청, 중복 시간표 등 의사결정에 중요한 상태를 색상/텍스트로 명확히 전달합니다.

5. 이수 진단 통합
	- 신청 학점과 졸업 이수 기준을 같은 흐름 안에서 확인할 수 있도록 자가진단 탭을 제공합니다.

## 4. 주요 기능

### 4.1 강좌 탐색 및 신청
- 실시간 검색: 강좌명, 교수명, 학과명, 이수구분(category) 기준 즉시 필터링
- 강좌 카드 UI: 카테고리 태그 + 핵심 메타데이터 표시
- 신청/취소 흐름: 모달 확인 기반의 명시적 액션 처리
- 정원 초과 시 대기신청 처리 및 대기 번호 반영
 - 장바구니 탭: 강좌를 장바구니에 담고 순서를 조정한 뒤 일괄신청으로 순차 처리

### 4.2 시간표 연동 및 충돌 방지
- 신청 강좌 자동 시간표 반영
- 기존 신청 과목과의 시간 중복 검사
- 마우스 오버 시 시간표 하이라이트(타깃/중복/신청 강좌 포커스)

### 4.3 학점 요약 및 자가진단
- 신청 학점(신청 + 대기) 실시간 집계
- 졸업 기준 대비 취득/수강/잔여 학점 확인
- 카테고리별 세부 진단 조회 버튼 제공

### 4.4 사용성 보조 기능
- 다크모드 토글 및 상태 저장(localStorage)
- 좌/우 패널 리사이즈 핸들
- 상단 서버 시간 표시(정각 10초 전 시각 강조)
- 강의 상세 정보/강의계획서 모달 조회
- 장바구니 순서 조정(▲/▼) 및 일괄신청 플로우
- 모바일 뷰 토글: 430px 폭의 세로형 레이아웃과 시간표 보기/숨기기 토글 제공

## 11. 한계 및 향후 개선

- 현재는 목업 데이터 기반 프로토타입이며 실제 학사 시스템 API와 연동되지 않습니다.
- 향후 개선 방향:
  - 실시간 시간표 프리뷰 강화
  - 다중 필터(이수구분/학년/요일/학점) 고도화
  - 사용자 테스트 기반 UI 반복 개선
